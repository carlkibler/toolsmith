Current positioning notes:

- Installer that detects coding tools and asks before integrating with each one.
- Auto-updating coding-agent efficiency kit.
- Self-audits impact with hard stats: token savings, edit payload reduction, avoided full-file reads/writes, and tool usage over time.
- Possible future umbrella for useful, attribution-preserving pieces from Dirac, cozempic, and tokenlean.

## 2026-05 naming exploration

Decision for now: **leave Toolsmith alone**. It remains flexible enough for the current shape: anchored edit primitives, MCP/Pi adapters, setup/update/doctor, usage audit, live harnesses, and tripwire nudges.

If renaming later, evaluate candidates against these product promises:

- exact, hash-anchored file reads and edits
- safe mutation: validate before writing, fail loudly on drift
- agent behavior steering: make the better tool path shorter than native muscle memory
- cross-client harnessing: MCP, Pi.dev, Claude/Codex/Gemini/OpenCode/etc.
- measurable leverage: token savings, missed opportunities, adoption scans

### Strongest shortlist

1. **Hashrail** — best sharp/technical candidate: hash-validated, rail-guided, safe path.
2. **AnchorKit** — clearest if the core remains anchored editing primitives plus adapters.
3. **Agentrail** — best if the product becomes primarily about steering agents across many clients.
4. **Writeguard** — best if trust/no-clobber is the headline.
5. **Linesmith** — best continuity candidate if we want to keep Toolsmith's craft feel.
6. **Triprail** — best if tripwire/adoption becomes the central product story.

Working instinct: keep **Toolsmith** unless a future positioning pass decides the main promise is no longer "a practical kit agents reach for" but instead "safe rails for agent file edits" (**Hashrail** / **Agentrail**) or "line-level craft and exact edits" (**Linesmith**).
